# 16-Week Calendar (Equal Track Allocation)

Each week has 4 sessions:

- A: Coding (90m)
- B: System Design (90m)
- C: AI Interview (90m)
- D: Review/Adapt (30m)

## Weeks 1-4 (Baseline Build)

- Coding: arrays/strings, sliding window, trees, graphs basics
- System Design: requirements, API shape, storage basics, caching basics
- AI: LLM fundamentals, prompt structure, basic RAG architecture, eval basics
- Goal: establish baseline score and error taxonomy

## Weeks 5-8 (Depth)

- Coding: DP, graph traversal variants, heaps, binary search on answer
- System Design: partitioning, consistency models, reliability and failure handling
- AI: chunking/retrieval strategies, hallucination controls, latency/cost tradeoffs
- Goal: reduce critical errors by 30% in each track

## Weeks 9-12 (Pressure + Communication)

- Coding: hard mixed sets, optimization and edge-case robustness
- System Design: large-scale scenarios, bottleneck analysis, tradeoff justification
- AI: agentic workflows, tool use, eval framework design, safety constraints
- Goal: stable pass-level with clear communication in all tracks

## Weeks 13-16 (Interview Simulation)

- Coding: interview-like mixed random set, strict time limits
- System Design: ambiguous prompts with follow-up pressure
- AI: trend-heavy prompts (agent reliability, RAG quality, governance)
- Goal: final readiness signal and next-cycle specialization plan

## Checkpoints

- End Week 4: baseline report
- End Week 8: error reduction report
- End Week 12: consistency report
- End Week 16: readiness decision
