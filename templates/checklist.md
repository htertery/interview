# Session Checklist

Use this before and after every session.

## Pre-Session

- [ ] Prompt selected
- [ ] Timebox defined
- [ ] Success criteria noted
- [ ] No external hints/tools (unless explicitly planned)

## During Session

- [ ] Problem framing done in first 3-5 minutes
- [ ] Assumptions stated clearly
- [ ] Complexity and tradeoffs explained
- [ ] Edge cases explicitly handled
- [ ] Communication stayed structured and concise

## Post-Session Scoring

- [ ] Score assigned (0-100)
- [ ] Critical misses identified
- [ ] Root cause tagged (knowledge, speed, clarity, execution)
- [ ] One follow-up drill chosen

## Track-Specific Checks

### Coding

- [ ] Correctness validated on representative cases
- [ ] Time and space complexity justified
- [ ] Code is readable and testable

### System Design

- [ ] Requirements clarified
- [ ] High-level design communicated early
- [ ] Data model, scaling, and failure handling covered
- [ ] Tradeoffs justified with constraints

### AI Interview

- [ ] Architecture includes retrieval/model/eval boundaries
- [ ] Hallucination/safety concerns addressed
- [ ] Cost, latency, and quality tradeoffs discussed
- [ ] Evaluation metric(s) proposed
